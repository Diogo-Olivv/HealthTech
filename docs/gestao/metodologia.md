# Metodologia de Desenvolvimento

O projeto segue um **Scrum Adaptado** com sprints semanais.

## Ciclo Semanal

- **Sprint semanal:** Backlog priorizado com duração de 1 semana.
- **2 reuniões por semana:** Cada reunião é registrada seguindo o [template de ata](atas_reuniao/template_ata.md).
- **Retrospectiva:** 15 min ao final da sprint: o que funcionou / impedimentos / próximas tarefas.
- **Entrega semanal:** Ao fim de cada semana, o grupo preenche o [formulário de entrega](entregas/entrega_semanal/entrega_template.md).

## Fluxo por Feature

1. Integrante escolhe ou cria a issue que vai trabalhar na semana.
2. Abre branch seguindo os [padrões de Git](../desenvolvimento/padroes/git.md).
3. Implementa (pair programming com outro integrante é opcional).
4. Abre PR, outro integrante revisa e faz o merge na `main`.
5. Atualiza a documentação da feature no MkDocs.

**Dedicação esperada:** 10h semanais por integrante.
